## 專案願景

**StudySignal 致力於成為台灣學生的 AI 英語學習夥伴——在對話、語音與真實情境中練習英文，並將學習訊號（Signals）轉化為可追蹤的進步軌跡。**

## V1.0

**目標：** 完成英文學習核心產品，可作為對外 **Beta 公開測試** 或 **小規模試用** 的穩定版本。

### 預計完成項目

| 項目 | 現況 | V1.0 目標 |
|------|------|-----------|
| **Tutor 完整對話** | 🟡 基礎已完成 | 多輪穩定、含圖作業流暢、錯誤處理與 loading UX 完善 |
| **拍照分析** | 🟡 Vision API 已有 | 作業／課本一鍵分析；結果與 Signals 整合；Tools OCR 流程上線 |
| **發音分析** | 🟡 部分完成 | 語音提交完整評分、練習引導、與 TTS 練習字詞串接 |
| **文法分析** | ✅ 基礎已有 | 提示詞與 UI 優化；錯誤類型分類更清晰 |
| **單字分析** | ✅ 基礎已有 | 詞彙建議、同義詞、例句品質提升 |
| **AI 學伴** | ⚪ 規劃中 | 主動追問、學習目標記憶、個人化語氣（仍遵守「不直接給答案」） |

### V1.0 技術與產品交付

- **Production 部署**：正式 HTTPS 網域、環境變數與金鑰管理策略。
- **Signals 穩定化**：分析結果格式固定、邊界案例（空輸入、超長對話）處理完善。
- **平板 UX 打磨**：Composer、麥克風、相機、分析按鈕流程順暢無阻。
- **品質門檻**：`npm run build` 通過；核心流程平板實機回歸測試清單。

### V1.0 明確不包含

- 家長帳號與每日報告（留 V1.5）
- 多 AI Provider 切換（留 V2.0）
- 國文／數學／自然科目（留 V2.0）

## V1.5

**目標：** 從「單人學習工具」進化為「可持續使用的學習產品」，支援家長參與與學習動機機制。

### 預計完成項目

| 項目 | 說明 |
|------|------|
| **家長模式** | 家長只讀檢視學習摘要、Signals 趨勢；需帳號／角色與權限 |
| **每日報告** | 當日對話次數、弱項、建議練習；可推播或 Email（可選） |
| **學習歷程** | 對話、分析、翻譯紀錄跨 session 保存與回顧 |
| **任務系統** | AI 依程度派發每日／每週任務，與 Tutor、練習字詞串接 |
| **徽章** | 完成任務、連續學習、發音進步等視覺化獎勵 |
| **成就系統** | 長期里程碑（如累積對話輪數、Signals 分數提升） |

### V1.5 架構前提

- **後端資料持久化**：使用者、學習紀錄、任務狀態（資料庫或 BaaS）。
- **身分驗證**：學生／家長帳號綁定與隱私合規（未成年人資料保護）。
- **事件模型**：為 V2.0 Learning Analytics 預留結構化事件紀錄。

## V2.0

**目標：** 成為可擴展的 **AI 學習平台**——多科目、多模型、可分析、可營運。

### 學習分析與視覺化

| 項目 | 說明 |
|------|------|
| **Learning Analytics** | 使用頻率、弱項分佈、進步曲線；支援教學與家長決策 |
| **能力地圖** | grammar / vocabulary / fluency / pronunciation 長期輪廓視覺化 |

### 多 AI Provider 架構

| 項目 | 說明 |
|------|------|
| **多 AI Provider** | 伺服器端統一抽象，客戶端僅呼叫 `/api/*` |
| **Provider Registry** | 以環境變數切換，無需修改 UI 與業務邏輯 |
| **Gemini** | Google 模型接入（Chat / Vision / 可選 TTS） |
| **Claude** | Anthropic 模型接入 |
| **Grok** | xAI 模型接入 |
| **本地模型** | Ollama 等本機或內網部署；降低成本與資料外送 |

**切換原則：** `AI_PROVIDER=openai|gemini|claude|grok|local`，僅改設定與 adapter，不動 `StudySignalHome` 等前端流程。

### 多語言與多科目

| 項目 | 說明 |
|------|------|
| **多語言** | 介面與 Tutor 語言策略可配置 |
| **國文** | 閱讀、寫作、文言文等 Tutor 與分析模組 |
| **數學** | 解題引導（引導式，非直接給答案）、步驟檢核 |
| **自然** | 概念解釋、實驗圖解、科學詞彙 |

### V2.0 營運與擴展

- 模型路由策略（成本／品質／延遲）可配置。
- 機構版或學校試點（班級、教師儀表板為延伸方向）。
- 正式 SLA、監控、錯誤追蹤與用量控管。

## 長期目標

**StudySignal 成為 AI 學習平台。**

具體方向：

1. **全科目 AI 學伴**：英語為起點，擴展至國文、數學、自然及更多領域。
2. **訊號驅動的個人化學習**：Signals 不只是單次回饋，而是長期能力地圖與學習路徑的基礎。
3. **家長與教育現場橋接**：家長看得懂進步、教師（未來）可參考學習數據，而不取代親師溝通。
4. **開放且可替換的 AI 底座**：Provider-agnostic，隨技術與成本演進自由選擇模型。
5. **台灣學生情境優先**：繁體中文輸入輔助、平板 HTTPS、課本作業拍照、英美發音等在地需求內建於產品 DNA。

願景一句話延伸：**讓每一位學生的學習訊號清晰可見，並在 AI 陪伴下持續進步。**

## 開發原則

以下原則貫穿 Beta → V1.0 → V1.5 → V2.0，不因功能擴張而妥協。

| 原則 | 說明 |
|------|------|
| **Mobile First** | 以 Android 平板與手機為主要設計與測試目標；觸控、safe area、IME 必備。 |
| **HTTPS First** | 區域網路與正式環境皆 HTTPS；麥克風與 Secure Context 為硬性前提。 |
| **Provider-agnostic** | 客戶端只認 `/api/*` 契約；第三方 API 僅在伺服器 adapter；環境變數切換模型。 |
| **Low Cost First** | 瀏覽器 TTS、優先 Web Speech API、Whisper 作後備；避免不必要雲端費用。 |
| **AI 引導學習** | Tutor 與分析引導思考、改寫、練習，而非代寫作業答案。 |
| **不直接給答案** | 幫我找英文僅輔助表達；解題與作業流程強調過程與理解。 |
| **文件優先** | 架構、Roadmap、安裝、Git、專案狀態置於 `docs/`；重大變更同步更新。 |
| **Git 管理** | 功能分支、有意義的 commit、遠端備份；合併前 build 通過。 |
