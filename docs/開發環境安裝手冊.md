# StudySignal 開發環境安裝手冊

本手冊說明如何在 Windows 11 上建立 StudySignal 首頁專案（`studysignal-homepage`）的完整開發環境，並以 **HTTPS** 模式在本機與 Android 平板上測試麥克風等需要 Secure Context 的功能。

---

## 1. 開發環境需求

在開始之前，請確認已安裝並可使用下列工具。

### Windows 11

StudySignal 開發環境以 **Windows 11** 為主要測試平台。請使用已更新至最新版本的系統，並確保可執行 PowerShell 或 Windows Terminal。

### Node.js LTS

專案使用 **Node.js LTS**（長期支援版）執行 Next.js 與 npm 指令。

- 下載：https://nodejs.org/
- 安裝後在終端機驗證：

```bash
node --version
```

此指令會顯示目前安裝的 Node.js 版本，應為 LTS 系列（例如 `v22.x.x`）。

```bash
npm --version
```

此指令會顯示 npm（Node 套件管理器）版本，用於安裝專案相依套件。

### Git

用於從 GitHub 複製（clone）專案與版本控制。

- 下載：https://git-scm.com/download/win

```bash
git --version
```

此指令會確認 Git 已正確安裝並可在命令列使用。

### Cursor

**Cursor** 為本專案建議使用的整合開發環境（IDE），內建 AI 輔助功能。

- 下載：https://cursor.com/
- 安裝後可使用 **File → Open Folder** 開啟專案資料夾。

### Chrome / Edge

建議使用 **Google Chrome** 或 **Microsoft Edge** 進行本機開發測試。兩者皆支援開發者工具，且對 Web Speech API、麥克風權限的除錯較為方便。

- Chrome：https://www.google.com/chrome/
- Edge：Windows 11 通常已預裝

## 2. 下載專案

### Clone GitHub repository

在欲放置專案的目錄開啟終端機（PowerShell 或 Windows Terminal），執行：

```bash
git clone https://github.com/hsiaotsunjen-dotcom/studysignal.git
```

此指令會從 GitHub 將遠端儲存庫完整複製到本機，建立 `studysignal` 資料夾（若你的本機資料夾名稱為 `studysignal-homepage`，可能是先前手動重新命名或不同 clone 路徑，兩者皆可，以實際資料夾為準）。

進入專案目錄：

```bash
cd studysignal
```

若你的資料夾名稱不同，請改為實際路徑，例如：

```bash
cd studysignal-homepage
```

### Open project in Cursor

1. 啟動 **Cursor**。
2. 選擇 **File → Open Folder**（或 **檔案 → 開啟資料夾**）。
3. 選取剛才 clone 下來的專案根目錄（內含 `package.json`、`src/` 等）。
4. 確認左側檔案總管可看到專案結構。

建議在 Cursor 內建終端機（**Terminal → New Terminal**）執行後續 npm 指令，工作目錄會自動位於專案根目錄。

## 3. 安裝相依套件

在專案根目錄執行：

```bash
npm install
```

### 說明

- `npm`：Node.js 的套件管理工具。
- `install`：依 `package.json` 與 `package-lock.json` 下載並安裝所有相依套件到 `node_modules/`。
- 首次 clone 或 `package.json` 更新後都需執行一次。
- 安裝內容包含 **Next.js**、**React**、**TypeScript**、**Tailwind CSS** 等開發與建置所需套件。

安裝成功後，專案根目錄應出現 `node_modules/` 資料夾（通常不需手動修改）。

## 4. HTTPS 開發環境

StudySignal 的語音輸入（麥克風）與部分瀏覽器 API 在開發階段必須透過 **HTTPS** 存取，不可僅依賴區域網路的 HTTP。

### 為什麼需要 HTTPS

瀏覽器將 **HTTPS** 視為安全連線。許多敏感 API（包含 `getUserMedia` 麥克風）僅在 **Secure Context** 下可用。若使用 `http://192.168.x.x:3000` 在平板上開啟，瀏覽器會判定為非安全環境，麥克風可能完全無法使用。

### 為什麼麥克風需要 Secure Context

**Secure Context** 表示頁面在下列任一情境載入：

- `https://` 開頭的網址
- `http://localhost` 或 `http://127.0.0.1`（本機例外）

在 Android 平板上以區域網路 IP（例如 `http://192.168.164.142:3000`）開啟時，**HTTP 不算 Secure Context**，因此 `navigator.mediaDevices.getUserMedia` 可能不存在或呼叫失敗。開發時應使用 **`https://<LAN-IP>:3000`**。

### HTTP 與 HTTPS 的差異

| 項目 | HTTP | HTTPS |
|------|------|--------|
| 連線 | 明文 | 加密（TLS） |
| 預設埠（開發） | 本專案 `npm run dev` 使用 3000 | 本專案 `npm run dev:https` 使用 3000 |
| Secure Context（區域 IP） | 否 | 是（需有效或已接受的憑證） |
| 麥克風在平板上 | 通常不可用 | 可用（需使用者授權） |

本專案提供兩種開發指令：

- `npm run dev`：HTTP，僅適合本機 `localhost` 快速預覽。
- `npm run dev:https`：**HTTPS**，為平板測試麥克風的**標準開發方式**。

> **重要：** 在平板上測試時請一律使用 `npm run dev:https`，不要使用 `npm run dev`。

## 5. Install mkcert

**mkcert** 可在本機產生受開發機信任的 TLS 憑證，並可將根憑證安裝到 Android 平板，減少憑證警告。

### Install

**方式一：Chocolatey（建議，需已安裝 Chocolatey）**

```bash
choco install mkcert
```

**方式二：手動下載**

請參考官方安裝說明：https://github.com/FiloSottile/mkcert#installation

安裝後驗證：

```bash
mkcert -version
```

### Create local CA

在終端機執行（通常只需執行一次）：

```bash
mkcert -install
```

此指令會在本機建立 mkcert 的根憑證機構（CA），並將其加入 Windows 的受信任儲存區，使本機瀏覽器信任由 mkcert 簽發的開發用憑證。

### Trust local CA

- **Windows（開發機）：** 執行 `mkcert -install` 後即已信任。
- **Android 平板（選用，建議）：** 將 mkcert 根 CA 匯出到平板並安裝為使用者憑證，可減少「連線不安全」警告。根 CA 路徑可查詢：

```bash
mkcert -CAROOT
```

依平板型號，路徑通常為：**設定 → 安全性 → 加密與憑證 → 安裝憑證**（實際選單名稱可能因廠牌而異）。

> 若尚未安裝 mkcert，仍可使用 `npm run dev:https`（Next.js 內建自簽憑證），但平板每次可能需手動接受憑證警告。

## 6. Generate HTTPS certificates

在專案根目錄、已安裝 mkcert 的前提下執行：

```bash
npm run setup:dev-https
```

### 說明

- 此指令執行 `scripts/setup-dev-https.mjs`。
- 會偵測本機 **LAN IPv4**（例如 `192.168.164.142`），並為 `localhost`、`127.0.0.1`、`::1` 及所有 LAN IP 產生憑證。
- 需要系統已安裝且可用 `mkcert` 指令；若未安裝，腳本會提示安裝方式並結束。

### 產生的檔案

憑證會寫入專案根目錄下的 `certificates/`（此資料夾通常不提交至 Git）：

| 檔案 | 說明 |
|------|------|
| `certificates/dev-key.pem` | TLS 私鑰 |
| `certificates/dev-cert.pem` | TLS 憑證（含 localhost 與 LAN IP） |

之後執行 `npm run dev:https` 時，若偵測到上述檔案存在，會自動使用 mkcert 憑證，而非 Next.js 預設自簽憑證。

若更換 Wi-Fi 網路導致電腦 LAN IP 改變，請重新執行：

```bash
npm run setup:dev-https
```

## 7. Start development server

在專案根目錄執行：

```bash
npm run dev:https
```

### 說明

- 執行 `scripts/dev-https.mjs`，以 **HTTPS** 模式啟動 Next.js 開發伺服器。
- 預設監聽 **port 3000**。
- 若存在 `certificates/dev-*.pem`，會使用 mkcert 憑證；否則使用 Next.js 實驗性自簽憑證。

### 預期成功輸出（範例）

使用 mkcert 憑證時，終端機可能顯示類似：

```
[dev:https] Using certificates/dev-*.pem (mkcert, includes LAN IP)
[dev:https] Local:   https://localhost:3000
[dev:https] Network: https://192.168.164.142:3000
   ▲ Next.js 15.x.x
   - Local:        https://localhost:3000
   - Network:      https://192.168.164.142:3000
 ✓ Ready in 3s
```

未使用 mkcert、改為自簽憑證時，可能顯示：

```
[dev:https] Using Next.js built-in self-signed certificate
[dev:https] Tip: run `npm run setup:dev-https` once for trusted LAN certs
```

看到 **`Ready`** 且 **Local / Network** 皆為 **`https://`** 開頭，即表示 HTTPS 開發伺服器已正常啟動。

本機瀏覽器可開啟：

```
https://localhost:3000
```

## 8. Android tablet testing

### Connect to same Wi-Fi

1. 確認開發用 **Windows 電腦**與 **Android 平板**連上**同一個 Wi-Fi**。
2. 確認電腦防火牆允許區域網路存取 port **3000**（必要時在 Windows 防火牆允許 Node.js）。

### Open https://\<LAN-IP\>:3000

1. 在電腦上執行 `npm run dev:https` 後，記下終端機顯示的 **Network** 網址，例如：

```
https://192.168.164.142:3000
```

2. 在平板瀏覽器（Chrome）網址列輸入該網址（將 IP 換成你終端機顯示的實際 LAN IP）。

查詢本機 IP（若終端機未顯示）可在 Windows 執行：

```bash
ipconfig
```

在「無線區域網路介面卡」或使用中介面下找到 **IPv4 位址**。

### Accept certificate warning

若使用自簽憑證或平板尚未安裝 mkcert 根 CA，瀏覽器可能顯示「連線不是私人連線」或類似警告：

1. 點選 **進階** 或 **Advanced**。
2. 選擇 **繼續前往**（Proceed）以載入頁面。

僅在開發環境對自己的開發機 IP 這樣做；正式環境應使用有效 CA 簽發的憑證。

### Verify microphone permission

1. 在平板開啟 StudySignal Talk 頁面。
2. 點選麥克風（Talk）相關按鈕。
3. 瀏覽器應跳出 **麥克風權限**請求，請選擇 **允許**。
4. 開發模式下可觀察 Secure Context 診斷面板（若已啟用）：
   - `isSecureContext` 應為 `true`
   - `mediaDevices` / `getUserMedia` 應存在

若權限遭拒，請至 **Chrome → 網站設定 → 麥克風**，找到對應 IP 並改為允許。

## 9. Common problems

### ERR_SSL_PROTOCOL_ERROR

**現象：** 平板開啟 `https://192.168.x.x:3000` 出現 `ERR_SSL_PROTOCOL_ERROR`。

**常見原因：** port 3000 上跑的是 **HTTP**（`npm run dev`），而非 **HTTPS**（`npm run dev:https`）。瀏覽器期待 TLS，伺服器卻回應明文 HTTP。

**處理方式：**

1. 停止目前 dev server（終端機按 `Ctrl + C`）。
2. 改執行：

```bash
npm run dev:https
```

3. 確認終端機顯示的網址為 `https://` 開頭。

### Microphone unavailable

**現象：** 麥克風按鈕無反應，或診斷顯示 `isSecureContext: false`。

**常見原因：**

- 使用 `http://192.168.x.x:3000` 而非 HTTPS。
- 未授予麥克風權限。
- 平板與電腦不在同一網路。

**處理方式：**

1. 一律使用 `https://<LAN-IP>:3000`。
2. 確認已執行 `npm run dev:https`。
3. 在瀏覽器網站設定中允許麥克風。
4. 重新載入頁面後再試。

### HTTPS not running

**現象：** 無法連線，或只能以 HTTP 開啟。

**處理方式：**

- 確認指令為 `npm run dev:https`，不是 `npm run dev`。
- 確認終端機無錯誤訊息，且出現 `Ready`。
- 若曾修改過環境，重新執行 `npm install` 後再啟動。

### Wrong LAN IP

**現象：** 平板無法連線，或連到錯誤機器。

**處理方式：**

1. 在開發機執行 `ipconfig`，確認目前 IPv4。
2. 與 `npm run dev:https` 輸出的 **Network** 網址比對。
3. 若 IP 已變更（換 Wi-Fi），更新平板書籤，並視需要重新執行 `npm run setup:dev-https`。

### Port 3000 occupied

**現象：** 啟動時提示 port 3000 已被占用。

**處理方式（Windows PowerShell）：**

查詢占用程序：

```powershell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue | Select-Object OwningProcess -Unique
```

結束舊的 dev server 程序（將 `<PID>` 換成上一步查到的數字）：

```powershell
Stop-Process -Id <PID> -Force
```

然後重新執行：

```bash
npm run dev:https
```

若問題與快取有關，可停止伺服器後刪除 `.next` 再啟動（僅開發快取，不影響原始碼）：

```bash
# PowerShell
Remove-Item -Recurse -Force .next
npm run dev:https
```

## 10. Daily startup checklist

每日開始開發前，建議依序完成：

- [ ] **Start Cursor** — 開啟 Cursor，載入 StudySignal 專案資料夾。
- [ ] **Pull latest code** — 在專案根目錄執行：

```bash
git pull
```

從遠端取得最新程式碼，避免與團隊版本脫節。

- [ ] **npm install (if needed)** — 若 `git pull` 後 `package.json` 有變更，或 `node_modules` 異常，執行：

```bash
npm install
```

- [ ] **npm run dev:https** — 啟動 HTTPS 開發伺服器：

```bash
npm run dev:https
```

確認終端機顯示 `https://localhost:3000` 與 `https://<LAN-IP>:3000`。

- [ ] **Test microphone** — 在本機或平板上開啟 HTTPS 網址，測試麥克風權限與 Talk 功能是否正常。
- [ ] **Begin development** — 確認環境無誤後開始功能開發。
